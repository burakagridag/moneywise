## US-031: More Tab — Settings Ana Ekranı

**Epic:** Sprint 6
**As a** MoneyWise user
**I want to** see a structured settings menu under the More tab with all configuration
entry points organized into sections
**So that** I can navigate to any app setting without searching through a flat list

### Acceptance Criteria

```gherkin
Scenario: More tab opens settings menu with all sections
  Given I am on any tab
  When I tap the "More" tab (4th tab)
  Then the MoreScreen opens at route "/more"
  And the AppBar displays "Settings" in the center
  And the app version is shown in the top-right corner (textTertiary, non-tappable)
  And I see Section 1 containing: Help, Feedback, Rate it, Remove Ads.
  And I see Section 2 "Trans." containing: Transaction Settings, Repeat Setting
  And I see Section 3 "Category/Accounts" containing: Income Category Setting,
      Expenses Category Setting, Accounts Setting, Budget Setting
  And I see Section 4 "Settings" containing: Backup, Passcode, Main Currency Setting,
      Sub Currency Setting, Alarm Setting, Style, Language Setting

Scenario: Each settings row navigates to its target screen
  Given I am on the MoreScreen
  When I tap "Style"
  Then I navigate to the StyleScreen at route "/more/style"
  When I go back and tap "Language Setting"
  Then I navigate to the LanguageScreen at route "/more/language"
  When I go back and tap "Main Currency Setting"
  Then I navigate to the CurrencyScreen at route "/more/currency-main"

Scenario: Section separators visually group settings
  Given I am on the MoreScreen
  Then each section is visually separated by a bold background separator
  And each row displays: left icon (32x32, textSecondary), center title (textPrimary),
      optional subtitle (textTertiary, footnote), right chevron ">" or value
```

### Edge Cases
- [ ] Very long section list — verify the full list is scrollable without bottom content cut off by the bottom nav bar
- [ ] Rows with subtitles (e.g., "Transaction Settings" with subtitle "Monthly Start Date, Carry-over Setting, Period, Other") must not overflow on small screens (iPhone SE)
- [ ] "Passcode" row must show current status ("ON" / "OFF") on the right as a live value, not static text
- [ ] "Main Currency Setting" row subtitle must reflect the currently selected currency (e.g., "EUR(€)")
- [ ] Offline — entire MoreScreen is local-only; all rows must render and be tappable without network

### Test Scenarios for QA
1. All four sections render on iOS and Android
2. Every row navigates to the correct route
3. Passcode row status reflects actual passcode state (ON/OFF)
4. Currency subtitle updates after changing currency and returning to MoreScreen
5. Scroll to bottom of list — no overlap with the bottom tab bar

### UX Spec
See `docs/specs/SPEC-018-more-screen.md`

### Estimate
M (3–5 days)

### Dependencies
- US-003 (go_router + 4-tab navigation)
- US-032 (StyleScreen — navigated from MoreScreen)
- US-033 (CurrencyScreen — navigated from MoreScreen)
- US-029 (BudgetSettingScreen — already exists, linked from MoreScreen)

## US-032: Dark / Light / System Tema Seçimi

**Epic:** Sprint 6
**As a** MoneyWise user
**I want to** choose between Dark, Light, and System (auto) themes from the Style screen
**So that** the app appearance matches my preference or my device's system setting

### Acceptance Criteria

```gherkin
Scenario: StyleScreen displays three theme options with current selection highlighted
  Given I navigate to More > Style
  Then I see three radio-style options: "System Mode", "Dark Mode", "Light Mode"
  And the currently active theme option has a brand-color filled circle on the right
  And inactive options show a border-only circle
  And the screen background and text already reflect the current theme

Scenario: Selecting Dark Mode applies the dark theme immediately
  Given I am on the StyleScreen
  And the current theme is "Light Mode"
  When I tap "Dark Mode"
  Then the theme switches to dark immediately without requiring an app restart
  And the selected option circle becomes brand-color filled
  And the setting is persisted in the "theme" key of the Settings KV store as "dark"
  And after I close and reopen the app, the dark theme is still active

Scenario: Selecting System Mode follows the device appearance setting
  Given I am on the StyleScreen
  When I tap "System Mode"
  Then the theme setting is saved as "system"
  And when the device is in dark mode, the app uses the dark theme
  And when the device is in light mode, the app uses the light theme
  And switching the device appearance (without closing the app) updates the app theme reactively

Scenario: Theme persists across app restarts
  Given I selected "Light Mode" and closed the app
  When I reopen the app
  Then the app launches in Light Mode
  And the StyleScreen still shows "Light Mode" as selected
```

### Edge Cases
- [ ] Theme switch while a modal (e.g., Add Transaction) is open — modal must also update instantly without visual glitch
- [ ] System theme switch at midnight (automatic dark/light transition) — app must respond without requiring restart
- [ ] Cold app launch with "system" setting and device in dark mode — no flash of incorrect theme on startup (avoid FOUC)
- [ ] Theme token values (bgPrimary, textPrimary, etc.) must be sourced from AppTheme ThemeExtension, not hardcoded — verify with code review
- [ ] On Android, system theme change while app is in background — verify theme is correct when app is resumed

### Test Scenarios for QA
1. Select each of the three modes, verify immediate visual change on iOS
2. Select each of the three modes, verify immediate visual change on Android
3. Select Light Mode, close app, reopen — verify Light Mode persists
4. Set System Mode on iOS, switch device to dark/light in Settings — verify reactive update
5. Set System Mode on Android, toggle dark mode — verify reactive update
6. Verify Settings KV store value ("dark" / "light" / "system") after each selection

### UX Spec
See `docs/specs/SPEC-019-style-screen.md`

### Estimate
S (1–2 days)

### Dependencies
- US-031 (MoreScreen — entry point)
- US-010 (theme system from Sprint 1 — toggle persistence already partially exists; this story replaces any ad-hoc toggle with the StyleScreen UI)

## US-033: Para Birimi Seçimi (Ana Para Birimi)

**Epic:** Sprint 6
**As a** MoneyWise user
**I want to** select my main currency from a searchable list of ISO 4217 currencies
**So that** all monetary values across the app are displayed in my preferred currency

### Acceptance Criteria

```gherkin
Scenario: CurrencyScreen opens with current main currency highlighted
  Given I navigate to More > Main Currency Setting
  Then the CurrencyScreen opens at route "/more/currency-main"
  And a search bar is visible at the top
  And the list shows currencies with: flag + currency code + currency name + symbol per row
  And the currently selected currency row has a brand-color check mark on the right
  And the currently selected currency text is brand-colored

Scenario: Searching filters the currency list in real time
  Given I am on the CurrencyScreen
  When I type "TRY" in the search bar
  Then the list filters to show only rows matching "TRY" or "Turkish lira"
  And if I clear the search, the full list is restored
  And if no currency matches the search term, an empty state message "No currencies found" is shown

Scenario: Selecting a currency updates the main currency immediately
  Given I am on the CurrencyScreen
  And the current main currency is "EUR"
  When I tap "TRY — Turkish Lira — ₺"
  Then the row gets a brand-color check and the EUR row's check is removed
  And the setting is persisted in the "mainCurrency" key of the Settings KV store as "TRY"
  And all monetary displays across the app (Trans. tab, Accounts tab, Stats tab) now use "₺"
  And the More tab subtitle for "Main Currency Setting" updates to "TRY(₺)"

Scenario: Currency change does not alter stored transaction amounts
  Given there are existing transactions stored with amounts in EUR
  When I change the main currency to USD
  Then the stored transaction amounts in the database remain unchanged (no conversion is applied)
  And the currency symbol displayed changes to "$"
  And existing transactions are displayed with their original numeric value but the new symbol
```

### Edge Cases
- [ ] Currency list has 150+ entries — verify smooth scrolling performance on low-end Android (no jank)
- [ ] Search with special characters (e.g., "£", "€") — must match by symbol as well as by code/name
- [ ] Changing currency while Add Transaction modal is open — the modal's currency symbol must update if it is still open
- [ ] Offline — the full currency list is loaded from the local `currencies` table (seeded at install); no network required
- [ ] Currency code exactly 3 characters (ISO 4217 constraint) — UI must not truncate codes

### Test Scenarios for QA
1. Open CurrencyScreen — currently selected currency is highlighted
2. Search "EUR" — filtered to single result
3. Search "lira" — shows TRY (and any other "lira" currencies)
4. Select TRY — all app money displays switch symbol to ₺
5. Restart app — TRY remains selected
6. Verify stored transaction amounts are unchanged after currency switch
7. Empty search result state renders correctly

### UX Spec
See `docs/specs/SPEC-020-currency-screen.md`

### Estimate
S (2–3 days)

### Dependencies
- US-031 (MoreScreen — entry point)
- US-004 (Drift DB — settings_table KV store and currencies_table)

## US-034: Transaction Arama (Metin Bazlı)

**Epic:** Sprint 6
**As a** MoneyWise user
**I want to** search my transactions by typing a description, note, or category name
**So that** I can find a specific past transaction without manually scrolling through the list

### Acceptance Criteria

```gherkin
Scenario: Tapping the search icon opens the search modal
  Given I am on the Transactions tab (any sub-view)
  When I tap the magnifier icon in the AppBar (left side)
  Then a SearchModal slides in (or a search bar replaces the AppBar)
  And an auto-focused text input is shown
  And the results area is initially empty with a "Type to search..." placeholder

Scenario: Typing returns matching transactions in real time
  Given the SearchModal is open
  When I type "lunch"
  Then the results list updates within 300ms to show all transactions whose
      description, note, or category name contains "lunch" (case-insensitive)
  And each result row shows: category emoji + category name + account name + date + amount
  And the matching substring is visually highlighted in the result row

Scenario: Search returns no results
  Given the SearchModal is open
  When I type a string that matches no transactions (e.g., "xyznotfound")
  Then the results area shows an empty state illustration and the text
      "No transactions found for 'xyznotfound'"
  And no error is thrown

Scenario: Tapping a search result navigates to the transaction
  Given I have search results visible
  When I tap a result row
  Then the SearchModal closes
  And I am taken to the TransactionDetailScreen (or the Daily View scrolled to that transaction)
  And the back button returns me to the Transactions tab

Scenario: Clearing the search field resets results
  Given I have typed "lunch" and results are shown
  When I tap the clear (X) button inside the search field
  Then the input is cleared
  And the results area returns to the "Type to search..." placeholder state
```

### Edge Cases
- [ ] Very short query (1 character) — search must execute but may have performance implications; debounce input by at least 300ms before querying
- [ ] Query with leading/trailing whitespace — must be trimmed before querying
- [ ] Transaction with empty description and empty note — must not appear when searching non-empty strings, but must appear under an explicit empty search if supported
- [ ] Large dataset (10,000+ transactions) — Drift full-text search or LIKE query must return results without visible lag; verify index usage
- [ ] Special characters in search input (e.g., %, _, quotes) — must be safely parameterized to prevent SQL injection or unexpected results
- [ ] Offline — search works entirely locally; no network dependency
- [ ] iOS keyboard appearance pushes content — results list must be above the keyboard (use `resizeToAvoidBottomInset`)

### Test Scenarios for QA
1. Search "food" — all Food category transactions appear
2. Search by partial note text — matching transactions appear
3. Search with no matches — empty state renders
4. Tap result — navigates to correct transaction
5. Clear search — results reset
6. Search with 10,000 transactions in DB — results appear within 500ms
7. Test on iOS (keyboard behaviour) and Android

### UX Spec
See `docs/specs/SPEC-021-search-modal.md`

### Estimate
M (3–4 days)

### Dependencies
- US-003 (TransactionsScreen — AppBar search icon entry point)
- US-004 (Drift DB — transactions table with indexes)
- US-010 (TransactionAddEdit — transaction detail navigation target)

## US-035: Transaction Filtreleme (Kategori, Tarih Aralığı, Tip)

**Epic:** Sprint 6
**As a** MoneyWise user
**I want to** filter my transaction list by transaction type (income/expense/transfer),
category, and/or date range
**So that** I can view a focused subset of my transactions for analysis or record-keeping

### Acceptance Criteria

```gherkin
Scenario: Tapping the filter icon opens the FilterModal
  Given I am on the Transactions tab
  When I tap the filter icon (3 horizontal lines + dot) in the AppBar (right side)
  Then a FilterModal opens (bottom sheet slide-up)
  And I see filter sections: "Type" (Income / Expense / Transfer chips),
      "Category" (multi-select list), "Date Range" (from/to date pickers)
  And all filters are initially unset (no active filters)

Scenario: Applying a Type filter shows only matching transactions
  Given the FilterModal is open
  When I tap "Expense" under Type
  And I tap "Apply"
  Then the FilterModal closes
  And the Transactions tab shows only Expense transactions for the current month
  And a filter-active indicator (e.g., badge or highlighted icon) appears on the filter icon
  And the income/expense summary bar still reflects totals for the full month
    (or is updated to reflect filtered scope — to be confirmed by UX spec)

Scenario: Applying a Category filter narrows the list further
  Given I have already applied an "Expense" type filter
  When I re-open the FilterModal
  And I select categories "Food" and "Transport"
  And I tap "Apply"
  Then only Expense transactions in the Food or Transport categories are shown
  And both Type and Category filters appear active

Scenario: Applying a Date Range filter overrides the month navigator
  Given the FilterModal is open
  When I set From Date: 2026-04-01 and To Date: 2026-04-15
  And I tap "Apply"
  Then only transactions between April 1 and April 15, 2026 (inclusive) are shown
  And the month navigator is visually locked or hidden while a date range filter is active

Scenario: Clearing all filters restores the full list
  Given one or more filters are active
  When I open the FilterModal and tap "Clear All"
  Then all filter selections are removed
  And the Transactions tab shows the full list for the current month
  And the filter-active indicator on the filter icon is removed
```

### Edge Cases
- [ ] All three filter dimensions active simultaneously — the query must apply all as AND conditions
- [ ] Date range where From Date is after To Date — "Apply" must be disabled with an inline validation message
- [ ] Filter applied to a month with zero transactions after filtering — empty state must render (not a blank screen)
- [ ] Category filter list with 30+ custom categories — the list inside FilterModal must be scrollable
- [ ] Filter state must survive the app going to background and resuming (use provider state, not ephemeral widget state)
- [ ] Filters must not persist across app restarts unless explicitly saved (filters are session-scoped)
- [ ] "Transfer" type filter: transfer transactions have no category — category filter section must be disabled or hidden when only Transfer type is selected

### Test Scenarios for QA
1. Filter by Type = Expense only — verify only expense rows appear
2. Filter by Category = Food — verify only Food transactions appear
3. Filter by Date Range (Apr 1–15) — verify only transactions in range appear
4. Combine Type + Category + Date Range — verify AND logic
5. Clear All — verify full list restores
6. Invalid date range (From > To) — Apply button disabled
7. Filter with zero results — empty state shown
8. Filter state survives app backgrounding (iOS/Android)

### UX Spec
See `docs/specs/SPEC-022-filter-modal.md`

### Estimate
M (3–4 days)

### Dependencies
- US-034 (Search modal, shares TransactionsScreen AppBar design)
- US-003 (categories — for multi-select list)
- US-004 (Drift DB — query composition for combined filters)

## US-036: Arama ve Filtre Kombinasyonu

**Epic:** Sprint 6
**As a** MoneyWise user
**I want to** apply both a text search and one or more filters at the same time
**So that** I can pinpoint a specific transaction with maximum precision when I remember
only partial details

### Acceptance Criteria

```gherkin
Scenario: Text search and active filters are applied simultaneously
  Given I have an active filter: Type = "Expense", Category = "Food"
  When I open the search bar and type "lunch"
  Then the results show only Expense + Food transactions whose description,
      note, or category name contains "lunch"
  And the filter-active indicator on the filter icon remains visible
  And the search result count reflects the combined filter

Scenario: Changing a filter while search text is entered updates results immediately
  Given I have typed "market" in the search bar
  And I open the FilterModal
  When I add Category = "Groceries" and tap Apply
  Then the search results narrow to only Groceries transactions containing "market"

Scenario: Clearing search text while filters are active preserves the active filters
  Given I have Category = "Transport" filter active and have typed "taxi" in search
  When I clear the search field
  Then the results revert to showing all Transport transactions (filter still active)
  And the search bar returns to the "Type to search..." placeholder

Scenario: Clearing all filters while search text is entered preserves search results
  Given I have a Type = "Income" filter active and "salary" in the search bar
  When I open FilterModal and tap "Clear All"
  Then the active filter is cleared
  And the results list shows all transactions (across all types) containing "salary"
```

### Edge Cases
- [ ] Combined query (LIKE + filter conditions) must use a single Drift query, not two separate in-memory passes — verify with code review
- [ ] When both search and filters return zero results, the empty state message must reference both dimensions: "No transactions match 'lunch' in Food expenses"
- [ ] Search + date range filter: date range should constrain searched results (not override the search)
- [ ] Performance: combined query on 10,000+ transactions must complete within 500ms

### Test Scenarios for QA
1. Set Expense + Food filter, search "lunch" — combined results correct
2. Clear search text — filter still active, full filtered list shown
3. Clear all filters — only search results remain
4. Combined query with zero results — empty state message describes both constraints
5. Performance test: 10,000 transactions, combined search + filter under 500ms

### UX Spec
N/A — covered by SPEC-021 (Search) and SPEC-022 (Filter); combination behavior is a
provider-layer concern, not a separate screen.

### Estimate
S (1–2 days — incremental on top of US-034 and US-035)

### Dependencies
- US-034 (text search)
- US-035 (filter modal)

## US-037: Bookmark Oluştur (Transaction'dan Şablon Kaydet)

**Epic:** Sprint 6
**As a** MoneyWise user
**I want to** save a filled-in Add Transaction form as a bookmark (template)
**So that** I can quickly re-use frequently entered transactions without re-filling
every field each time

### Acceptance Criteria

```gherkin
Scenario: Bookmark icon is visible on Add Transaction screen when form has content
  Given I am on the Add Transaction screen
  And I have entered Amount: 12.50, Category: "Food", Account: "Debit Card"
  Then the bookmark icon (star + list) in the AppBar top-right is tappable
  And tapping it opens a "Save as Bookmark" bottom sheet

Scenario: Saving a bookmark creates a record in the bookmarks table
  Given the "Save as Bookmark" bottom sheet is open
  And the bookmark name field is pre-filled with the category name (e.g., "Food")
  When I change the name to "Daily Lunch" and tap "Save"

Scenario: Bookmark with optional amount (blank amount field)
```

3. Save Transfer bookmark — both from/to accounts captured
4. Duplicate name warning appears and allows proceeding

## US-039: Bookmark Yönetimi (Liste, Düzenle, Sil)

**Epic:** Sprint 6
**As a** MoneyWise user
**I want to** view all my saved bookmarks, edit their details, and delete ones I no longer need
**So that** I can keep my bookmark list organized and up to date

### Acceptance Criteria

```gherkin
Scenario: Bookmark management screen is accessible from More tab
  Given I am on the MoreScreen
  When I tap "Bookmark Setting" (or the equivalent row — exact label TBD by UX spec)
  Then I navigate to the BookmarkManagementScreen
  And the screen lists all non-deleted bookmarks sorted by sortOrder, then useCount descending
  And each row shows: type chip (Income/Expense/Transfer) + name + amount or "—" + category + account

Scenario: Editing a bookmark updates its fields
  Given I am on the BookmarkManagementScreen
  When I tap the edit (pencil) icon on the "Daily Lunch" row
  Then a BookmarkEditModal opens pre-filled with all current bookmark fields
  And I can change: name, amount, category, account, note, description, type
  When I tap "Save"
  Then the bookmark record is updated in the DB
  And the list row reflects the new values immediately (reactive Drift stream)

Scenario: Deleting a bookmark removes it from the list
  Given I am on the BookmarkManagementScreen
  When I swipe left on the "Daily Lunch" row (or tap a delete icon)
  Then a confirmation dialog appears: "Delete 'Daily Lunch' bookmark?"
  When I confirm
  Then the bookmark is soft-deleted (isDeleted = true) in the DB
  And the row disappears from the list with an animated removal
  And any existing transactions that were created from this bookmark are NOT affected
      (bookmarkId on old transactions is kept as-is)

Scenario: Reordering bookmarks via drag-and-drop updates sortOrder
  Given I am on the BookmarkManagementScreen
  When I long-press and drag a bookmark row to a new position
  Then the bookmark's sortOrder is updated in the DB
  And the BookmarkPickerModal will respect the new order on next open
```

### Edge Cases
- [ ] Edit modal opened for a bookmark whose account has been deleted — Account field shows "(deleted account)" warning; saving is allowed but with the same deleted account reference (or user can re-select)
- [ ] Deleting last bookmark — empty state must appear: "No bookmarks. Add one from the Add Transaction screen."
- [ ] Bulk delete — if a multi-select mode is supported, deleting multiple bookmarks must be a single DB transaction (all-or-none)
- [ ] Bookmark name field in edit modal: max 50 characters (same as create)
- [ ] Rapid edit-save cycles — Riverpod state must not show stale data between edits
- [ ] Drag reorder: if the user drops back to the original position, no DB write should occur

### Test Scenarios for QA
1. Bookmark list shows all saved bookmarks correctly
2. Edit bookmark name and amount — changes reflect in list immediately
3. Edit bookmark type (Expense → Income) — type chip updates
4. Delete single bookmark — confirmation, then removal with animation
5. Delete last bookmark — empty state shown
6. Drag reorder — sortOrder persisted, picker modal reflects new order
